## Why

插件已经通过命令面板提供了丰富的批量操作能力，但高频操作在右键菜单中的可发现性不足，导致使用路径较长。引入与上下文匹配的右键菜单入口，可以显著降低编辑与文件管理场景下的操作摩擦。

## What Changes

- 在编辑器右键菜单中新增 File Cooker 分组入口，并按“选区操作”和“当前文件链接操作”进行子菜单组织。
- 在文件右键菜单中新增 File Cooker 分组入口，提供单文件操作，并将高风险操作放在更安全的位置。
- 引入上下文目标适配层，使右键菜单触发可复用现有 Action 与确认弹窗流程。
- 保留现有命令面板命令，确保向后兼容与使用习惯不受影响。

## Capabilities

### New Capabilities
- `context-menu-commands`：在 Obsidian 的编辑器右键菜单与文件右键菜单中，以分组且上下文感知的方式暴露 File Cooker 能力。

### Modified Capabilities
- 无。

## Impact

- 影响代码范围：插件初始化与命令/菜单注册相关模块（`main.ts`、`src/command/*` 及其 Action 分发衔接逻辑）。
- 用户体验影响：提升能力可发现性，缩短高频操作路径。
- 风险点：菜单噪音控制、上下文判断准确性、右键触发与原命令触发的一致性。
